问卷系统功能逻辑设计建议

本问卷系统定位为支持复杂条件判断、身份验证、循环展开、分组统计与精细化数据分析的高级问卷系统。系统应在保留基础问卷能力的同时，支持复杂业务场景，如自评、互评、投票、报名、分组填报、按身份差异化作答、按对象逐项评价等。

一、基础题型与题目约束

问卷设计界面支持以下基础题型：

单选题

多选题

单行文本题

多行文本题

数值滑杆题

循环块（特殊结构型题目）

其中各题型支持如下约束：

1. 单选题

可设置必填或选填

可设置选项默认统计方式

可参与条件逻辑、联合判定、跳转逻辑、分组统计等

2. 多选题

可设置必填或选填

可设置最少选择项数

可设置最多选择项数

可参与条件逻辑、联合判定、跳转逻辑、分组统计等

必填多选题自动生成一个列表对象，表示当前答卷者在该题中选择的值集合

3. 单行文本题

可设置必填或选填

可限制最大/最小字数

可限制最大/最小词数

可参与文本类条件逻辑

可用于格式判定、关键词判定、长度判定等

4. 多行文本题

可设置必填或选填

可限制总字数

可限制总行数

可参与文本类条件逻辑

可用于补充说明、原因追问、开放性反馈等场景

5. 数值滑杆题

可设置必填或选填

可设置最小值与最大值

可设置步进值

可参与数值判定、联合判定、自动计算、统计分析等逻辑

二、题目级条件逻辑

每道题都应支持配置条件逻辑。条件逻辑用于根据前序信息决定后续题目、选项、统计归属及展示方式。系统应保留你原本的逻辑主干，并在其上补充更完整的判断能力。

1. 基础判定对象

条件逻辑可基于以下内容进行判断：

单选题答案

多选题答案

数值题答案

单行文本题答案

多行文本题答案

当前答卷者身份属性

当前循环对象属性

某组题的整体特征

某列表对象的内容、长度、交集关系等

2. 基础判定方式

系统应支持以下通用判定：

等于 / 不等于

大于 / 小于 / 大于等于 / 小于等于

位于某范围内 / 不在某范围内

为空 / 不为空

包含 / 不包含

命中某一项 / 某几项

满足全部条件 / 任一条件 / 不满足条件

3. 条件组合逻辑

在你已有联合判定逻辑的基础上，应补充：

AND（且）

OR（或）

NOT（非）

多层嵌套组合条件

条件组中至少满足 N 个

条件组中至多满足 N 个

条件组中恰好满足 N 个

这类补充不改变你原有的联合限制结构，而是让其表达能力更完整。

三、数值题与选择题的增强判定逻辑

你原本已经提出数值题与单选/多选题可进行大小、联合、总和、计数等判定，这部分非常关键，应保留并适当补强。

1. 数值题判定

数值题支持：

与固定值比较大小

与另一数值题比较大小

与一组数值题的平均值、总和、最大值、最小值比较

是否位于指定区间

是否高于/低于某阈值

2. 多题联合数值限制

系统支持对同类多道数值题做联合判定，例如：

总和等于 / 不等于 / 大于 / 小于某值

总和不能超过某值

总和不能不足某值

平均值高于 / 低于某值

最大值 / 最小值满足某条件

极差满足某条件

并补充支持：

超过某数值的题目数至少为多少

超过某数值的题目数至多为多少

低于某数值的题目数恰好为多少

满足某条件的连续题目数达到多少

某组题中满足某条件的项数达到某阈值

3. 单选/多选题集合逻辑

在你原有单选/多选判定基础上，建议补充：

是否包含任一指定选项

是否同时包含全部指定选项

是否只包含某几项

是否与某组选项存在交集

是否完全不包含某组选项

两题所选集合是否相同

两题所选集合是否有重叠

某组题中命中某类选项的题数达到多少

这仍然属于你已有“单选/多选题可尽量加入类似联合判定逻辑”的扩展，不破坏原结构，只是补得更细。

4. 相对比较逻辑

在绝对值判定之外，建议补充：

某题是否为一组题中的最大值 / 最小值

某题是否高于另一题

两题差值是否超过某阈值

某题是否为并列最大 / 并列最小

某组中某一项是否显著偏高 / 偏低

四、文本题条件逻辑

你原逻辑中对文本题的条件判断涉及较少，建议作为补充加入，但仍保持其从属于“题目条件逻辑”的整体框架。

1. 文本内容判定

单行文本题和多行文本题可支持：

是否为空

是否非空

文本长度大于 / 小于 / 位于某范围

词数大于 / 小于 / 位于某范围

行数大于 / 小于 / 位于某范围

是否包含某关键词

是否不包含某关键词

是否匹配某种格式

是否与另一文本题相同 / 不同

2. 文本逻辑触发用途

当文本满足特定条件时，可触发：

显示后续追问题

将该答卷单独列出展示

纳入某一特殊数据分组

作为异常样本单独展示

与其他题进行联合判定

五、条件触发后的逻辑动作

你原文中已经提出：当判定符合特定条件时，可以有多个逻辑，包括展示题目、单独展示、不纳入整体比例、划入数据分组等。这一部分是你的框架核心之一，应保留，并加入补充动作。

1. 原有逻辑动作

满足条件后，可执行：

展示某些特定题目

不展示某些特定题目

数据统计时单独展示

不划入整体比例

划入某数据分组

以分组方式统计数据比例或特征

2. 补充逻辑动作

在不改变你原有动作集合的前提下，建议增加：

跳过某段题目

跳转到某题或某题块

提前结束问卷

进入不同答题分支

给当前答卷打上某个标签

显示特定说明文本或提示文本

自动带出默认值或引用值

限制提交

要求补充说明后才可继续提交

将答卷标记为特殊样本、候选样本、异常样本或重点展示样本

这里的“跳转、结束、分支”是最重要的补充之一，它和“显示隐藏”不同，但能很好并入你原来的条件触发体系。

六、选项级逻辑

你原来主要是题目级逻辑，建议在不改变原有结构的前提下，补充“选项级逻辑”，作为题目逻辑的细化层。

1. 选项显示逻辑

某些选项可根据条件：

显示

隐藏

禁用

启用

2. 选项关系逻辑

可支持：

某选项与其他选项互斥

选择某选项后自动禁止选择某些其他选项

选择某选项后要求必须再选某些选项

选择“无”后不可再选其他项

某组选项必须至少选一个

某组选项最多选几个

3. 配额逻辑

建议补充：

某选项最多允许多少人选择

某组选项总名额限制

某身份组在某选项上的可选人数限制

名额满后选项自动不可选

这部分特别适合报名、投票、资源分配类场景。

七、去重逻辑与唯一性逻辑

你原有框架中已经有身份验证和循环对象场景，因此很有必要加入去重逻辑作为补充。

1. 题内去重

多个位置不得重复选择同一对象

一组分配题中不得重复分配同一对象

同一标签不得重复赋予同一对象

2. 循环过程中的去重

循环块中多个实例不得重复选择相同名单项

某循环对象不得被重复评价 / 重复投票 / 重复提名

某个名额分配中同一人不能被多次占用

3. 身份去重

同一身份仅允许提交一次

同一身份在某一问卷模块中仅允许作答一次

同一身份对同一对象仅允许产生一条有效评价

这部分不会破坏原有逻辑，而是让名单、循环、投票、互评这类功能更完整。

八、自动计算与引用逻辑

这一部分是补充功能，不替代你原本的判定逻辑，而是增强其可用性。

1. 自动计算逻辑

系统可支持某些字段或提示内容根据已有答案自动计算，例如：

某数值题自动等于前几题之和

自动计算剩余可分配数值

自动计算已使用名额 / 剩余额度

自动生成某些统计型中间值供判定逻辑使用

2. 引用逻辑

题目标题、说明、选项说明等内容中，可引用：

当前答卷者姓名、学号、组别等身份属性

当前循环对象属性

另一题的答案

某列表长度

某组题的计算结果

例如：

“请评价当前对象：XX”

“你还剩余 XX 分可分配”

“你刚刚选择了 XX，请继续说明原因”

这部分和循环块、身份验证、联合判定都能自然衔接。

九、循环逻辑

你提出的“循环块”是整体框架中非常有特色也非常核心的部分，应保留原意并做补充。

1. 循环块定义

循环块是一种特殊题目结构，用于按某一列表对象反复展开一组题目。
循环依据必须是当前问卷中已有的列表对象。

2. 列表对象

列表是问卷中可供循环、筛选、判定等逻辑使用的可迭代对象。一个问卷中可有多个列表对象。设计者可自行定义列表对象，列表可存储数字或字符串，但不能混存。

系统自动生成的列表包括：

当问卷启用身份验证功能并绑定某一名单后，系统自动加入多个名单列表对象，分别存储名单中各项属性信息。

每道必填多选题都会创建一个列表对象，表示当前答卷者在该题中选择的值集合。此类列表对象在不同答卷过程中不同。

3. 循环块的补充能力

在你原本循环逻辑基础上，建议补充以下能力：

对循环列表设置筛选条件

对循环列表设置排序方式

可排除某些循环对象

可只循环满足条件的对象

可设置循环次数上限

可设置空列表时的行为（如不显示/显示说明）

4. 当前循环对象判定

在循环块中，系统可支持特殊条件逻辑：

如果该题的对象是当前答卷者

如果该题的对象不是当前答卷者

如果当前循环对象满足某身份属性

如果当前循环对象命中某条件组

你原本提到“如果该题的对象是当前答卷者”，这一项应明确作为循环块中的特殊条件逻辑保留。

5. 循环块中的联合判定

你原本要求“循环逻辑中也应加入联合判定逻辑”，这一点非常合理。补充后可支持：

对某循环块中全部实例的答案做总和、平均值、最大值、最小值、计数等判定

对某些循环实例满足条件的个数进行限制

对循环过程中某对象是否已被选中过进行判定

对循环内部题目与循环外题目联合判定

对循环对象之间进行重复限制、配额限制、分组限制

十、身份验证与身份联动逻辑

你原有身份验证逻辑已经比较明确，应保留为全局功能之一。

1. 身份验证功能

设计者可为问卷勾选身份验证功能。
在全局中可存储若干名单，每个名单可存储多项自定义身份信息。
当设计者勾选身份验证后，可选择其中一个名单，仅允许名单中的人员作答。

2. 身份属性联动

身份验证启用后，可将身份信息用于：

条件逻辑判断

题目显示 / 隐藏

分组统计

循环块筛选

当前答卷者对象判定

标签生成

数据单独展示

名额限制与去重限制

3. 身份相关补充逻辑

在不改变原意的基础上，建议加入：

同一身份是否允许重复提交

同一身份是否允许参与某个循环对象的多次评价

同一身份是否只能进入某些特定分支

身份属性与题目答案联合判定

身份冲突或身份矛盾答卷的单独展示

十一、统计方式与统计逻辑

你已经明确提出“每道题都应可以设置数据统计方式”，这部分很完整，应在原基础上补强而非替换。

1. 单选/多选题统计

单个单选/多选题可设置：

统计比例（默认）

统计频数

仅统计选择最多的几项

仅统计选择最少的几项

排序展示

分组后分别统计比例或频数

并建议补充：

Top N / Bottom N 展示

某组选项合并统计

某类选项单独统计

不同分组间交叉对比统计

2. 数值题统计

数值题可设置统计以下一个或多个指标：

平均值（默认）

最大值

最小值

中位数

标准差

和

极差

并建议补充：

方差

分位数

区间分布

有效样本数

缺失率

超阈值项数统计

3. 文本题展示逻辑

虽然文本题不适合常规比例统计，但可支持：

原文列表展示

按条件筛选展示

按长度分类展示

含某关键词的样本单独展示

与其他条件联合筛选后单独展示

4. 条件统计隔离与分组展示

当某题或某组条件勾选了以下逻辑时：

满足条件的答卷者单独列出展示

满足条件的一组答卷者组合在一起展示

满足条件的数据不划入整体比例

满足条件的数据划入某分组中展示

则系统应在统计分析时自动按这些规则处理。

5. 异常样本与特殊样本展示

作为补充，建议增加：

逻辑矛盾样本单独展示

极端值样本单独展示

低完成度样本单独展示

命中重点条件的样本单独展示

这仍可视作你原本“单独展示 / 分组展示”逻辑的细化补充。

十二、随机化与抽取逻辑（补充项）

这一项属于第三优先级补充，可加可不加，但如果要做得更完整，值得保留为可选能力。

系统可支持：

题目顺序随机

选项顺序随机

从题库中随机抽取若干题

从某一题组中按条件随机抽题

循环对象顺序随机

这类逻辑适用于测评、公平性控制、减少顺序偏差等场景。

十三、分值、权重与异常逻辑（补充项）

这一部分同样属于增强型能力，可作为补充加入，不改变原问卷逻辑主干。

1. 分值与权重

可支持：

单选/多选选项赋权

数值题折算分值

某组题加权求和

某些题反向计分

自动形成维度分、总分

2. 异常与矛盾判定

可支持：

前后答案矛盾时单独展示

与身份信息冲突时单独展示

互相排斥的条件同时命中时标记异常

极端作答模式标记为特殊样本

命中异常规则时限制提交或要求补充说明

这部分很好地补充了你原有“单独列出展示”和“分组统计”的逻辑用途。

总体总结

在保留你原有逻辑结构的前提下，这套问卷系统的完整功能框架可以概括为：

以基础题型为作答载体

以条件逻辑和联合判定为核心控制方式

以身份验证和列表对象为动态数据来源

以循环块为复杂场景展开机制

以分组统计、单独展示、统计隔离为结果分析机制

并补充跳转分支、文本判定、选项级逻辑、去重逻辑、配额逻辑、自动计算、引用、随机化、分值与异常判定等增强能力

这样整理后，你原来的逻辑主线并没有变，仍然是：

题型 → 约束 → 条件判定 → 逻辑动作 → 循环展开 → 身份联动 → 数据统计

只是每一层都补得更完整、更接近“全场景高级问卷系统”。
